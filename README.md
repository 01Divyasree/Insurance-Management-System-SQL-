# Insurance-Management-System-SQL-
A database-driven project designed to efficiently manage insurance operations including policy management, customer records, claims processing, and premium tracking. This system demonstrates strong SQL skills through structured schema design, data relationships, and advanced query implementation.

**🔍 Project Overview**

The Insurance Management System is built using SQL to simulate real-world insurance workflows. It helps in organizing and managing data related to customers, policies, agents, and claims in a structured and scalable manner.

**⚙️ Key Features**
Customer registration and profile management
Policy creation, renewal, and tracking
Claim request and approval management
Premium calculation and payment tracking
Agent and policy assignment system
Data integrity using constraints and relationships

**🧠 Concepts Used**
Relational Database Design (ER Model)
Normalization (1NF, 2NF, 3NF)
Primary Keys & Foreign Keys
Joins (INNER, LEFT, RIGHT)
Aggregate Functions
Subqueries & Nested Queries
Views and Indexing

**🗂️ Database Tables**
Customers
Policies
Agents
Claims
Payments

**🛠️ Tech Stack**
SQL (MySQL / PostgreSQL)
Database Management System

**📊 Sample Use Cases**
Retrieve active insurance policies for a customer
Track pending and approved claims
Calculate total premium collected
Identify top-performing agents

**✅ Conclusion**
The Insurance Management System demonstrates effective use of SQL in designing and managing a structured database for real-world insurance operations. It improves data organization, ensures accuracy, and simplifies tasks like policy management, claim processing, and premium tracking. Overall, the project highlights strong database design skills and practical application of SQL concepts.
